================================================================================
WORKFLOW RULES
================================================================================

1. CHECKOUT:
   git checkout [branch]  # codex/experimental, openclaw/experimental, etc.

2. READ (from canonical):
   # Always read from src/nexus_os/ for source code
   cat src/nexus_os/bridge/server.py

3. WRITE (own branch only):
   # Code changes go to your workspace
   vim .codex/my_feature.py
   git add .codex/my_feature.py
   git commit -m "codex: Add my_feature experimental"

4. COPY (from canonical):
   # Copy canonical files to experiment locally
   cp src/nexus_os/bridge/server.py .codex/server_experiment.py

5. PR (merge to main):
   # When ready, create PR from your branch to main
   # Review by main implementer (SPECI)
   # Merge after approval
