### [arXiv:2602.12194] Systematic Study of Malicious Tool Behaviors in LLM Agents
- **Relevance:** ASI02 — CIA triad taxonomy for malicious tools
- **Key insight:** First systematic study. Proposes CIA-based taxonomy (confidentiality/integrity/availability). Shows existing detection (VirusTotal, MCPScan) has HIGH false negative rates. MalTool: auto-synthesis of malicious tools via coding LLMs. Key finding: existing heuristic methods fail.
- **Link:** https://arxiv.org/pdf/2602.12194
- **Action:** 🔴 READ — the CIA taxonomy maps directly to your `risk_level` field design on SkillDefinition. Use it to define the risk classification schema.

### [arXiv:2601.12449] AGENTRIM: Detecting and Mitigating Tool-Driven Agency Risks
- **Relevance:** ASI02 — direct implementation reference
- **Key insight:** Framework that assigns risk labels (low/high) to tools via policy-dependent function L(·). Produces partition T = TL ∪ TH consumed by online orchestrator. Evaluates on AgentDojo. Explicitly references OWASP ASI02.
- **Link:** https://arxiv.org/pdf/2601.12449v1
- **Action:** 🔴 IMPLEMENT — the L(·) risk labeling function is exactly what your `skill_adapter.py` needs for `risk_level`. Port or adapt.

### [arXiv:2504.11703] Progent: Programmable Privilege Control for LLM Agents
- **Relevance:** ASI02 + ASI05 (unexpected execution)
- **Key insight:** Tool-level privilege control. Reduces attack success to 0% on AgentDojo/ASB/AgentPoison. 4 risk assessment principles: (1) read vs write/execute, (2) sensitive data handling, (3) action reversibility, (4) external exposure. Policy enforced before each tool call.
- **Link:** https://arxiv.org/html/2504.11703v2
- **Action:** 🟡 READ — 4 risk principles are directly applicable to your KAIJU auth pre-execution gate.
