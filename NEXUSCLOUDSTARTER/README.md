# Nexus OS A2A — Conversation Starter Pack

A comprehensive backup/restore system for the Nexus OS A2A multi-agent project conversation context.

## What This Is

This starter pack preserves the full state, workflow patterns, team dynamics, and project knowledge from the GLM-5 swarm integration conversation (40,405 lines, Reviews 1-4). Its purpose: when any agent (or new GLM-5 session) boots up, these files restore complete operational context — no context loss, no re-discovery.

|---|---|---|
| `00_PROJECT_STATE.md` | Current project status, phase, blockers, next actions | **ALWAYS FIRST** — boot priority |

| `08_GLOSSARY.md` | Domain-specific terms and acronyms | When unsure about terminology |

## Boot Sequence

```
1. Read 00_PROJECT_STATE.md      → Know where we are
2. Read 04_WORKSTYLE.md          → Know how to communicate
3. Read 03_WORKFLOW.md           → Know what process to follow
4. Read relevant domain file     → Know the technical context
5. Check worklog.md              → Know the latest ground truth
```

## Maintenance Rules

- Update `00_PROJECT_STATE.md` after every significant milestone or decision
- Update `06_DECISION_LOG.md` when new decisions are made
- Update `07_PHASE_ROADMAP.md` when phases change or MVP scope shifts
- Never delete — append with dates. History is the backup.
